## The Problem With AI Paper Tools

If you've tried using AI to help write a research paper, you've hit the same wall I hit:

- **Citation hallucinations** — ~40% of citations are fake. The tool makes up plausible-sounding papers. You spend hours tracking down references that don't exist.
- **Zero novelty** — every tool asks "what's the gap in the literature?" the same way. You get the same obvious suggestions everyone gets.
- **Obvious AI text** — you paste it into a Humanizer detector, it flags half the sentences. You spend more time rewriting than you saved.

I wanted a system that solved all three problems at once. So I built one.

## The Architecture

Sisyphus Academica is a self-coordinating swarm of 20+ specialized agents. Here's how it works:

### Phase 0: Voice Calibration

The system learns the author's voice from 2-3 paragraphs of their published writing. Sentence length distribution, vocabulary level, paragraph patterns, transition style. Every writer agent uses this profile to match the author's voice — not generic academic prose.

### Phase 1: Literature Review

5 parallel scouts hit arXiv, Semantic Scholar, CrossRef, and OpenAlex simultaneously. 500+ papers returned in minutes. Deduplicated, structured, citation-graphed.

### Phase 1.5: The Novelty Engines (The Moat)

This is the core differentiator. Six agents that think from completely different cognitive frames:

1. **The Contrarian** — Inverts every well-established claim in the field
2. **The Cross-Pollinator** — Imports solutions from 15 distant fields (astrodynamics, epidemiology, music theory, immunology...)
3. **The Assumption Excavator** — Finds unstated assumptions and tests what breaks if they're false
4. **The Counterfactual Generator** — Rewrites the field's history without the most-cited papers
5. **The Paradox Sifter** — Cross-references every "Limitations" section to find ignored contradictions
6. **The Heretic** — Reads only title+abstract, generates 50 wild hypotheses, then scores each against the actual paper to find the "haunting idea"

### Phase 5: Parallel Writing

5 writer agents work simultaneously (abstract, introduction, methods, results, discussion). Each has the Humanizer skill loaded as a token-level generation constraint:

- Zero em dashes (hard rule, paper fails if any appear)
- No significance inflation ("pivotal", "transformative")
- No AI vocabulary ("showcasing", "underscores", "testament")
- No synonym cycling, no rule of three, no filler phrases
- 41 patterns total, enforced at generation time

### Phase 6: Verification

3 modules run in parallel:

- **Citation Verifier** — every reference checked against Semantic Scholar + CrossRef. Must be in 2+ sources.
- **Statistical Auditor** — validates every p-value, effect size, sample size
- **AI-Pattern Detector** — scans every sentence for all 41 patterns

### Phase 7: Adversarial Review

10 personas read the paper independently. Each returns a score, strengths, weaknesses, questions, and a recommendation. All 10 must recommend acceptance.

### Phase 10: Formatting

LaTeX template matching, verified BibTeX generation, figure embedding, PDF compilation.

## Real Results

We ran the full pipeline on a paper about **Intent-Based Blockchain Execution via Agentic RAG and Swarm Consensus**:

- 100+ papers surveyed
- 50+ hypotheses generated by 6 novelty engines
- 4 revision rounds
- **13-page PDF, 26 verified citations, 3 figures**
- **0 AI-pattern violations, 0 em dashes, 0 hallucinated references**
- Review scores: Avg 4.6/10 → 8/10 after revisions

Full output in `examples/siren-paper/` on the repo.
